二、Maven

1. Maven简介

1.1 作用

  1. 提供了一套标准化的项目结构，如下图

    1. 目的：解决不同开发工具之间转换导致的项目结构不一致的问题

  1. 提供了一套依赖管理机制

    1. 目的：解决了依赖（jar，插件等）引入过程中的版本关系的管理问题

  1. 提供了一套标准化的构建流程（编译，测试，打包，发布......）

    1. 目的：以便于更好的构建开发项目

1.2 仓库

  • 本地仓库：本地电脑的一个目录，用于管理本地项目引入的依赖(jar)

  • 远程仓库（私服）：一般由公司团队搭建的私有仓库，我们这里默认使用阿里云的私服

  • 中央仓库：由Maven团队维护的全球唯一的仓库

  • 使用关系

    • 当项目中使用坐标引入对应依赖的jar之后

      • 首先查找本地仓库中是否有对应的jar包，如果有就直接引用

      • 如果没有，本地配置了私服，则从私服中查找，找到后会下载到本地仓库，然后引用

        • 如果本地私服也没有，则会去中央仓库查找，找到后下载到本地私服，再下载到本地仓库，然后引用

      • 如果没有配置私服，则直接去中央仓库查找，再下载到本地仓库，然后引用

3. IDEA配置Maven

3.1 当前项目（重点）

  • 选择IDEA --> file --> settings --> Build,Execution,Deployment --> Buiild Tools --> maven

  • 需要设置三个地方

    • Maven homedirectory: 选择maven的安装目录
    • User settings file:选择Maven的配置文件，如果不能修改需要勾选后面的Override
    • Local repository:选择Maven仓库地址，如果不能修改需要勾选后面的Override

3.2 新建项目（重点）

  • 不配置的话每次引进新项目都要配置
  • 选择IDEA --> file --> new Projects Setup (或者是 new Project settings) --> Settings for New Projects --> Build,Execution,Deployment --> Buiild Tools -- maven

  • 需要设置三个地方

    • Maven homedirectory: 选择maven的安装目录
    • User settings file:选择Maven的配置文件，如果不能修改需要勾选后面的Override
    • Local repository:选择Maven仓库地址，如果不能修改需要勾选后面的Override

 

4. IDEA创建Maven模块

4.1 坐标简介

  • 坐标：Maven 中的坐标是资源的唯一标识，帮助区分引入的jar包名称和版本信息

    • 使用坐标来定义项目或引入项目中需要的依赖(jar)

  • 坐标组成

    • groupId：定义当前Maven项目隶属组织名称（通常是域名反写，例如：mysql）

    • artifactId：定义当前Maven项目名称（通常是模块名称，例如 mysql-connector-java）

    • version：定义当前项目版本号（例如：5.1.46）

4.2 创建maven模块

  • 打开IDEA --> File --> Project Structure --> Modules --> + --> New Module --> Maven

5.3 maven的生命周期

  1. Maven 构建项目生命周期描述的是一次构建过程经历经历了多少个事件

  2. Maven 对项目构建的生命周期划分为3套

    1. clean：清理工作（了解）

    2. default：核心工作，例如编译，测试，打包，安装等（重点）

      1. 同一生命周期内，执行后边的命令，前边的所有命令会自动执行

    3. site：产生报告，发布站点等（了解）

3.整体生命周期，主要记住红色部分的关键步骤，其它仅做了解就好

三、MyBatis入门

1. MyBatis简介

  • MyBatis 是一款优秀的持久层框架，用于简化 JDBC 开发

    • 持久层：负责将数据到保存到数据库的那一层代码，常规来说JavaEE分为三层架构：表现层、业务层、持久层

    • 框架：框架就是一个半成品软件，是一套可重用的、通用的、软件基础代码模型

      • 作用：在框架的基础之上构建软件编写更加高效、规范、通用、可扩展

    • 简化：让几乎所有的JDBC代码可以省略，以及设置参数和获取结果集的工作更加高效

    • 在刚开始的时候可能觉得有很多配置和代码要写很繁琐，但是注意，这些都是一次性的工作，在后续新增方法和业务需求的时候，我们就会变得相对轻松很多了
